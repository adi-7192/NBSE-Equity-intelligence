# Phase 02.1 — UI Design Contract

> Visual and interaction contract for the workspace shell, navigation, settings, and personal integrations surfaces.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn |
| Preset | new-york |
| Component library | Radix via shadcn/ui |
| Icon library | lucide-react |
| Font | `Manrope` for interface text + `JetBrains Mono` for market/status monospace moments |

---

## Spacing Scale

Declared values:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Badge padding, inline icon gaps |
| sm | 8px | Compact label spacing, row gutters |
| md | 16px | Default control spacing and card padding baseline |
| lg | 24px | Section padding and sidebar item rhythm |
| xl | 32px | Main layout gaps and page section spacing |
| 2xl | 48px | Major section breaks and settings panel breathing room |
| 3xl | 64px | Page-level top/bottom spacing on desktop |

Exceptions: `12px` may be used for dense tabular rows or compact nav labels when needed.

---

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px / 15px | 500 | 1.55 |
| Label | 11px / 12px | 700 | 1.35 |
| Heading | 20px / 24px / 30px | 650-700 | 1.1-1.2 |
| Display | 36px | 700 | 1.0 |

Typography rules:
- `Manrope` carries the main interface, page headings, cards, and settings copy.
- `JetBrains Mono` is reserved for symbols, market metadata, timestamps, status chips, and compact navigation annotations.
- All-uppercase mono labels should be short and used sparingly as framing devices, not as paragraph text.

---

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#09090b` / graphite-black | Page background, workspace canvas |
| Secondary (30%) | `#111318` / charcoal panels | Sidebar, cards, elevated containers |
| Accent (10%) | `#d6a54a` / warm amber | Active nav, section highlights, selected states, key CTA emphasis |
| Destructive | `#d04f45` | Destructive actions and error states only |

Accent reserved for:
- active navigation
- primary call-to-action buttons
- key focus rings
- important status accents that are not market up/down semantics

Never use amber as a replacement for success/error market colors. Green and red remain reserved for financial movement or explicit system health semantics.

---

## Layout Contract

- The authenticated app uses a persistent left sidebar plus a slim top bar.
- Sidebar width should feel intentional on desktop and collapse into a sheet/drawer on smaller screens.
- The top bar should contain page context, lightweight workspace/system status, and the account menu rather than functioning as the primary navigation.
- Main content should sit inside a centered but wider workspace frame than the current `max-w-5xl` dashboard shell.
- Dashboard and settings pages should use modular panels with clear vertical rhythm rather than stacked anonymous blocks.

---

## Navigation Contract

- Primary sidebar items: Dashboard, Archives, Screener, Portfolio, Settings.
- `Admin` appears only for admin users.
- Future-facing destinations like Screener and Portfolio may exist as polished placeholders if their feature phases are not yet implemented.
- Active nav state must be visually obvious even in dense/dark layouts.
- Settings gets its own secondary left sub-navigation: Profile, Integrations, Security, Workspace Status.

---

## Settings & Integrations Contract

- Settings is a dedicated full page, not a modal.
- `Integrations` contains first-class cards/panels for Zerodha, OpenAI, Gemini, and Anthropic.
- Secrets are never shown in plaintext after save.
- Saved integrations show masked state and status metadata such as configured, last updated, and validation/connection state.
- Missing integrations should trigger polished setup guidance and empty states instead of broken controls.
- Zerodha UX should ask only for durable credentials required to make the app’s connection flow work.

---

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | `Save Settings` / `Connect Provider` / `Update Workspace` |
| Empty state heading | `Connection needed` |
| Empty state body | `This area is ready, but it needs your provider connection first. Open Settings to finish setup.` |
| Error state | `We couldn’t validate this connection right now. Check the saved credentials or try again in a moment.` |
| Destructive confirmation | `Remove connection`: `This disconnects the provider for your account until you reconnect it.` |

Tone rules:
- concise, operator-like, calm
- no consumer-app fluff
- clear next step in every empty/error state

---

## Motion & Interaction

- Use subtle reveal/stagger on major shell sections, not decorative micro-motion everywhere.
- Hover and focus states should be crisp and restrained.
- Loading states should prefer skeletons or structured placeholders over generic spinners when layout is known.
- Disconnected or unconfigured states should feel intentional, not like an exception path.
